# WX1 RKN Checker

WordPress-плагин для **пакетной асинхронной проверки сайтов** на наличие счётчиков аналитики, политики конфиденциальности и других признаков.

## Что проверяется

| Признак | Что ищем |
|---|---|
| **Google Analytics** | `analytics.js`, `gtag.js`, GTM, `UA-…`, `G-…` |
| **Политика конфиденциальности** | Ссылки/текст с `privacy`, `конфиденциальност` и т.п. |
| **Яндекс.Метрика** | `mc.yandex.ru/metrika`, `ym(`, `Ya.Metrika` |
| **Sape** | `sape.ru`, `__sape_`, `sape.js` |
| **LiveInternet** | `counter.yadro.ru`, `liveinternet.ru/counter` |
| **Комментарии** | `#comments`, `.comments-area`, Disqus, `comment_form` |

## Установка

1. Скопируйте папку `wx1-checker-rkn` в `/wp-content/plugins/`.
2. В административной панели WordPress перейдите в **Плагины** → **Установленные** и активируйте **WX1 RKN Checker**.  
   При активации автоматически создаются две таблицы в базе данных: `wp_wxrkn_sessions` и `wp_wxrkn_results`.

## Использование

1. В админ-меню появится раздел **RKN Checker**.
2. Вставьте список доменов (по одному на строку) в текстовое поле.
3. При необходимости скорректируйте настройки:
   - **Размер батча** — сколько сайтов проверяются параллельно (1–10).
   - **Задержка между батчами** — пауза в мс между группами запросов.
   - **Страниц на сайт** — главная страница + до N−1 внутренних страниц.
   - **Таймаут запроса** — лимит ожидания ответа от сайта (сек).
4. Нажмите **▶ Запустить проверку**.
5. Прогресс отображается в реальном времени; строки таблицы заполняются по мере завершения проверок.
6. Если признак **найден** — ячейка подсвечена **красным**.
7. По окончании проверки появляется кнопка **↓ Экспорт CSV** — скачивает результаты в формате CSV с BOM (корректно открывается в Excel).

## Архитектура

```
wx1-checker-rkn/
├── wx1-checker-rkn.php          # Точка входа, регистрация хуков
├── uninstall.php                # Удаление таблиц при деинсталляции
├── includes/
│   ├── class-database.php       # Создание таблиц, CRUD
│   ├── class-checker.php        # Логика обхода и проверки сайта
│   ├── class-admin.php          # Страница админки, подключение ресурсов
│   └── class-ajax.php           # AJAX-эндпоинты
└── assets/
    ├── css/admin.css            # Стили
    └── js/admin.js              # Клиентская логика (батчи, прогресс, таблица)
```

**Async-механизм:** клиентский (JS), без WP-Cron. Браузер последовательно посылает AJAX-запросы (`wp_ajax_wxrkn_check_domain`), сервер синхронно обходит сайт через `wp_remote_get()` и возвращает результат. Это гарантирует работу на любом хостинге без специальных настроек.

## Требования

- WordPress 5.6+
- PHP 7.4+

## Безопасность

- Все AJAX-запросы защищены nonce (`wp_create_nonce` / `check_ajax_referer`).
- Права доступа проверяются через `current_user_can('manage_options')`.
- Пользовательский ввод санируется (`sanitize_text_field`, `absint`).
- Вывод экранируется (`esc_html`, `esc_attr`, `esc_url`).
- SQL-запросы параметризованы через `$wpdb->prepare()`.
